# GarphRAG-Agent-Prediction 首页 UI 升级设计说明文档

## 一、 设计理念 (Design Philosophy)

由于项目删除了静态图片依赖，借此机会我们将整体 UI 从常规的明亮模式 (Light Theme) 彻底重构为 **Dark Cyber-Geometric (暗黑赛博几何)** 风格。这不仅解决了图片资源缺失的问题，更与项目本身的底层技术（知识图谱、百万级 Agent 模拟、平行世界推演）高度契合，强化了产品的“极客”、“硬核”与“未来感”。

### 1. 核心视觉特征
*   **极客暗黑风 (Terminal Dark)**：以深邃的纯黑 (`#050505`) 为底色，配以高对比度的极客荧光色（赛博橙 `#FF5722` 和 霓虹青 `#00E5FF`）。
*   **网格与几何体系 (Geometric & Grid)**：背景引入了细微的坐标网格线，所有容器采用硬朗的直角线框，传达出“数据演算”、“沙盒环境”的系统严谨性。
*   **HUD 终端交互 (HUD Console)**：右侧操作区被设计为类似黑客控制台（Terminal）的输入界面，顶部带有红黄绿控制点，上传区域和文本域融合了代码编辑器的视觉语言。

### 2. 动效与纯代码视觉 (Pure Code Visuals)
**不再依赖任何外部图片资源**，所有视觉和动画均通过 HTML/SVG 和纯 CSS 实现，确保极致的加载速度和全平台兼容性：
*   **核心节点动画 (Node Network SVG)**：利用 SVG 绘制了一个象征“现实种子”的中心节点，外围环绕着以不同速率公转的数据卫星，并配合脉冲 (`pulse`) 和扫描线 (`scanline`) 动画，完美隐喻了“多 Agent 围绕核心事件进行演算”的逻辑。
*   **Glitch 故障艺术**：标题和按钮 hover 状态融入了故障抖动（Glitch）动画，强化赛博朋克的失控与重组感。
*   **终端光标 (Blinking Cursor)**：文案中融入了闪烁的下划线光标，模拟终端输入效果。

## 二、 文案创意 (Copywriting Strategy)

文案整体从原先偏向商业介绍的口吻，转变为更具科幻感、控制感和技术深度的“系统指令”语言。

### 1. 主视觉区文案 (Hero Section)
*   **原标题**：上传任意报告，即刻推演未来
*   **新标题**：`INJECT REALITY / SIMULATE FUTURE` (注入现实切片 / 演算平行未来)
    *   *创意解析*：“上传”改为“注入 (Inject)”，将“报告”升华为“现实切片”，凸显用户作为“造物主”的上帝视角。

*   **原介绍**：即使只有一段文字，GarphRAG-Agent 也能基于其中的现实种子，全自动生成与之对应的至多百万级Agent构成的平行世界。通过上帝视角注入变量，在复杂的群体交互中寻找动态环境下的“局部最优解”
*   **新介绍**：基于核心认知图谱提取现实种子，毫秒级构建**百万级 Agent 平行世界**。通过全知视角注入变量干扰，在复杂的高维群体博弈中，动态坍缩出最优决策路径。
    *   *创意解析*：保留了核心功能点，但引入了“高维群体博弈”、“动态坍缩”等更具量子力学和复杂系统科学色彩的词汇。

*   **原 Slogan**：让未来在 Agent 群中预演，让决策在百战后胜出_
*   **新 Slogan**：`root@engine:~# 让未来在数字集群中预演，让决策在百战后胜出_`
    *   *创意解析*：保留原优秀金句，但在前置加入了 Linux root 用户的命令提示符 `root@engine:~#`，强化终端控制感。

### 2. 交互区文案 (Console Section)
将所有操作步骤化、指令化：
*   `[01] UPLOAD_KNOWLEDGE_BASE (PDF/MD/TXT)`：替代原“现实种子”。
*   `[02] INJECT_SIMULATION_PARAMETERS`：替代原“模拟提示词”。
*   `EXECUTE_SIMULATION` (执行推演) / `INITIALIZING_SEQUENCE...` (序列初始化中)：替代原“启动引擎”。

## 三、 功能性保障与响应式 (Functionality & Responsiveness)

1.  **逻辑零耦合**：新 UI 完全保留了 Vue 3 的 Composition API 逻辑。拖拽上传 (`handleDrop`)、Pinia 状态管理 (`projectStore.setPendingUpload`) 和 Vue Router 路由跳转 (`router.push`) 等核心功能未受任何影响。
2.  **流式布局**：通过 CSS Grid 和 Flexbox 的结合，界面在 1024px 以下会自动从双栏平铺转换为垂直堆叠，确保在不同尺寸的显示器和设备上都能保持极佳的阅读体验和操作性。
3.  **无图化优势**：彻底根除了对 `/static/image` 目录的依赖，消除了 404 图片报错风险，减小了打包体积。
